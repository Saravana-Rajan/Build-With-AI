# Coimbatore — Real, Recently Reported Civic Issues (2024–2026)

Purpose: ground the civic-tech demo in **real, sourced** Coimbatore problems (not only
synthetic seed data). These become realistic seeded complaints and talking points that a
Coimbatore MP / corporation councillor would immediately recognise.

Research window: 2024–2026. Sources are named news outlets, corporation/government records,
and (where flagged) peer-reviewed studies. Nothing here is fabricated; items that could not
be tied to a real source were dropped. Some issues were reported **city/ward-wide** rather
than for a single colony — these are flagged honestly rather than pinned to a fake locality.

### Sourcing caveats (for honesty in the demo, not for the audience)
- Items 1–2 (water crisis) are well-corroborated across outlets but were reported **ward/city-wide**, not for one named colony.
- Items 10 & 12 are from **research/peer-reviewed studies**, not news — included because they document specific, named-location pollution.
- Item 21's metro-rejection detail and items 19–20 rest partly on encyclopedic/infra records rather than a single news article; the underlying facts (metro rejection, CIBT stall, Ukkadam rebuild) are widely reported.
- The Hindu / New Indian Express blocked direct full-text fetch; headlines, outlets and dates were confirmed via aggregation, so a few URLs are section-level rather than exact article links.

## How to use in the demo

1. **Seed the complaint database with these as real historical tickets.** Each row maps
   cleanly to `area / ward → category → description → source`. Mix them with synthetic
   complaints so the feed looks lived-in, not demo-clean.
2. **Anchor talking points a Coimbatore MP would recognise.** Lead the pitch with the ones
   that are unmissable locally: the **Siruvani water crisis** (once-a-week supply), **Vellalore
   dump-yard + groundwater contamination**, **October 2024 flooding** (Race Course / Ukkadam
   subways), **Sanganur canal sewage & encroachment**, and **potholes / 1,847 roads to be
   re-laid**. These signal "you actually know this city."
3. **Show the workflow end-to-end on a real case.** E.g. take the *GN Mills flyover streetlight*
   or *Sidhapudur TNUHDB housing* item and walk a complaint from citizen report → routing to
   the right zone/department → resolution status. Real names make the routing believable.
4. **Cluster by sector for the analytics/dashboard view** — water, roads, drainage/flooding,
   garbage, street lights, housing, health, transport, encroachment — so the "insights" screen
   reflects Coimbatore's actual problem mix, weighted toward water + roads + drainage.
5. **Keep sources visible.** A small "reported in The Hindu / DT Next / TNIE" tag on seeded
   items makes the demo credibly grounded rather than invented.
